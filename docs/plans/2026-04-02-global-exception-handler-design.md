# Issue #19 设计文档：全局异常处理器优化

## 问题描述
当前全局异常处理器 `global_exception_handler` 直接暴露内部错误信息（如堆栈跟踪、文件路径），存在安全风险。

## 需求分析
1. ✅ 生产环境不返回堆栈跟踪、文件路径等敏感信息
2. ✅ 区分开发/生产环境：开发环境返回详细错误信息，生产环境返回通用错误提示
3. ✅ 所有错误响应统一格式
4. ✅ 添加测试用例验证不同环境下的错误返回
5. ✅ 完成后提交代码创建PR

## 实现方案

### Step 1: 修改 config.py - 增加环境配置
- 新增 `ENV` 配置项：`development`（开发）/ `production`（生产）
- 默认 `DEBUG` 根据环境自动设置

### Step 2: 新建 exceptions.py - 统一异常处理逻辑
- 定义统一的错误响应格式 `ErrorResponse`
- 根据环境判断返回的错误级别
- 清除 traceback 中的敏感路径信息

### Step 3: 修改 main.py - 注册全局异常处理器
- 移除原有的 `@app.exception_handler(Exception)`
- 使用新创建的异常处理逻辑

### Step 4: 添加测试用例
- 测试开发环境下返回详细错误信息
- 测试生产环境下不返回敏感信息
- 测试不同异常类型的处理

### Step 5: 提交代码并创建 PR
